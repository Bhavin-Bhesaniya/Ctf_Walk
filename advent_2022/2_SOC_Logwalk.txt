Log Files :-
 - Contain historical records of events and other data from application such Login attempts or failure, File access, Error 

 Contain :-
  1. Event timestamp                     (Date & Time)
  2. Service name that generating logfile(SSH that allows user to login into a system remotely)
  3. Actual event service log            (In event of fail authentication, what credentials tried and by whom(IP Address)) 
 
 
 Windows :- Built-in Event-Viewer 
  Application :-	Contain event related application to determine when service or application stopped and started and why | msoffice.exe
  Security    :-	System security like when user logs into system or access credential manager for passwords	           | cmnatic
  Setup       :-	All events related to system maintenance                                                               | Windows update logs stored here 
  System      :- System changes                                                                                         | System unexpectedly shutdown due power issue
  
      
 Linux(Ubuntu/Debian) :-
  - Software specific log file located within /var/log directory
  Ex :- ls -lah in /var/log
   
  Authentication     :- auth.log | Attempted either remotely or on system itself | Accessing another user after logging in
  Package Management :- dpkg.log | When installing new software(package), logged | Reverting and debugging change incase installtion cause unintended behaviour  
  Syslog             :- syslog   | Contain all event that happening in system background | crontabs executing service such as log rotation   
  Kernel             :- kern.log | Change in kernel from device such as networking equipment or physical device USB device | Firewalling registered
    
 
 Log Analysis Tools :-
  Splunk | Security Information and Event Management(SIEM)
  SIEM :- Dedicated but expensive to run | Takes time to configure and setup | Allow in-depth analysis but require proper knowledge to use
  
  grep :- 
   - Searching for given text in file   
   - grep will first assume file in your current directory unless tell it otherwise by providing file path
   
   -i :- Perform case insensitive search
   -E :- Searches using regex | 
   -r :- Search recursively   | search all of files in directory for this value
